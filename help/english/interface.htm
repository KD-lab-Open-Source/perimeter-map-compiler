Interface

Perimeter Map Compiler's interface consists of a main window and three additional panels, toggled by buttons. The Preview panel shows a 3-D model of the map. You can turn lighting on or off in the application preferences, as it significantly affects loading speed. The Details panel allows you to fine-tune your map. The Statistics panel shows you the average height an colour of your map. You may find this information useful when setting the zero height and fog colour of your map.

Note that the panels take time to generate the information they display. The Preview panel is the most expensive one, followed by the Statistics panel. Hiding a panel may speed up loading.

Perimeter Map Compiler uses a special architecture to respond faster to your actions. When you give it a command, such as to load a shrub, it pauses very briefly to extract the essential information you may need to make further actions, and then continues to do the time-consuming work in the background. You can safely continue carrying out any further actions while the application is working. The icon on the right side of the main window turns from green to orange when the application is busy.

If you keep Perimeter Map Compiler running while editing the map images, the application will watch the images, and update its panels when any of the images change. This allows you to see the result while working. Note that you have to save the image for the application to see the changes.

Menu

  • File
    • New Project: begin creation of a new map
    • Open Project: open a started map creation project
    • Pack Shrub: create and register a finished map
    • Unpack Shrub: open a finished map, to possible then install it
    • Install Map: install a project or a shrub
    • Manage Maps: installed maps list
    • Project Settings: this is where additional content can be toggled
    • Exit
  • Tools
    • Preferences: affect quality of the Preview, as well as Shrub generation
      • Texture colour tradeoff: The compiler can generate the texture either quickly, or well. This option affects shrub generation!
      • Mesh tolerance: The preview model can be created with different precision. The lower the tolerance, the more precise the model, but the more power a computer is needed to show it.
      • Zero-level colour: The zero-level is shown in the preview as a translucent square overlaying the map. This is its caolour.
      • Zero-level opacity: See above.
      • Shadows and lighting: Calculations of shadows and lighting significantly affects loading speed. Therefore, it can sometimes be better to disable them.
      • Display as map texture: The Compiler can show preview of different data - texture, zero-layer, or hardness.
    • Save Thumbnail: save a view of the map from above.
